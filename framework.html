The framework

Cross-task adaptation, made studyable

Every environment is built around an explicit, controllable latent. Because the shared structure is known to the evaluator, we can isolate exactly where an agent succeeds or fails.

Adaptation as meta-RL

An agent faces a sequence of N related tasks τ1z, τ2z, …, τNz drawn from a distribution p(τ | z) conditioned on a latent z ∼ p(z). The latent is the recurring structure the agent should acquire from experience — common knowledge, recurring patterns, reusable strategies. The agent never observes z directly; it must infer it from interaction.

Crucially, the policy parameters stay fixed across the sequence. Adaptation happens in-context: the agent's behavior on task τiz depends on the history it carries forward from earlier tasks.

at ∼ πφ( at | st, h<i, hi<t )     with meta-objective    J(φ) = 𝔼z 𝔼τ1:N [ Φ(G1, …, GN) ]

Here h<i is everything seen on tasks 1…i−1 (including inter-task feedback), and Φ chooses what aspect of adaptation we reward — e.g. cumulative return Σi Gi, final return GN, or improvement GN − G1. This induces an exploration–exploitation trade-off across tasks: information-seeking on early tasks is costly now but pays off on every later task.

Each environment factorizes along five axes

The latent is the central design choice. Around it, four further axes let us control difficulty and support independently.

🧬

Latent structure

The hidden regularity shared across tasks (e.g. a restricted target set, a best arm, a threshold). The object of adaptation.

⚙️

Core dynamics

The within-task decision problem — what the agent does each turn in isolation.

🎲

Task-specific variation

Instance-level differences across tasks (the particular goal, dynamics, or initial state).

💬

Prompt behavior

How much information about the latent is revealed up front — sets the agent's prior.

📨

Feedback type

What signal is added to the cross-task history after each task — governs how fast evidence accumulates.

⏳

Horizon N

How many tasks are available for adaptation.

Prompt conditions

The prompt sets how much the agent knows before it interacts. Every environment supports three.

PromptInformation revealedWhat it tests
no_infoNo indication tasks share any latent structure.Does the agent discover structure from experience alone?
some_infoA vague hint that some recurring structure may exist.Does it identify the cross-task regularity efficiently?
full_infoAn explicit description of the latent.Can it use latent information once given?

Feedback conditions

standard

Only a binary success/failure signal after each task — the realistic, sparse case.

information

The ground-truth outcome is revealed regardless of success, so evidence about the latent accumulates faster.

Measuring how agents adapt

Two agents with the same cumulative return can adapt very differently — one probing the latent, the other exploiting what it already knows. We decompose adaptation into two surrogate quantities and measure each with an agent-switching experiment: one agent explores tasks 1…K and builds the history, then a second agent inherits it and exploits tasks K+1…N.

Exploration efficiency

Do the agent's actions produce experience that is informative about the latent?

Exploitation efficiency

Does the agent convert that experience into better decisions on later tasks?
