Metrics & glossary

How we measure adaptation

Standard metrics tell you whether an agent improved. Ours also tell you how — by separating exploration from exploitation.

Return-sequence metrics

An agent's behavior over a sequence is summarized by its per-task returns G1, G2, …, GN, where Gi is the return on task i given the history carried from earlier tasks.

Cumulative return

Σi Gi

Rewards uniform competence across the whole sequence. This is the reward we optimize during Cross-Task RL.

Cross-task gain

(GN − G1) / G1

Directly isolates whether the agent gets better with experience. Read alongside final reward (floor/ceiling effects).

Final-task return

GN

Rewards final capability after the agent has had the full sequence to adapt.

Exploration vs. exploitation efficiency

Within one agent's trajectory the two are entangled: a later-task return reflects both how informative earlier interactions were and how well the agent acts on them. We disentangle them with an agent-switching experiment — measuring each as a relative quantity between two agents.

Pick a switch point K. One agent (the explorer) runs tasks 1…K and builds the cross-task history h<K. A second agent (the exploiter) inherits it and runs tasks K+1…N. The tail return is GY|X = Σi=K+1N Gi when πX explores and πY exploits.

ExploreEffX(πB | πA) = (GX|B − GX|A) / GX|A      ExploitEffX(πB | πA) = (GB|X − GA|X) / GA|X

ExploreEff holds the exploiter fixed at reference X and swaps the explorer; ExploitEff holds the explorer fixed and swaps the exploiter.
